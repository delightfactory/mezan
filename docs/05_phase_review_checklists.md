# Phase Review Checklists

## Product Review

- هل كل تدفق مكتوب بلغة المستخدم المصري؟
- هل يوجد مسار واضح للدخل غير المنتظم؟
- هل تم تحديد الفرق بين الرصيد الخام والرصيد الآمن؟
- هل تم تحديد الأولويات عند العجز؟
- هل توجد حالات تصحيح دون حذف؟

## Migration Review

- كل جدول عائلي يحتوي `family_id`.
- كل جدول مكشوف عليه RLS.
- كل policy تتحقق من `auth.uid()` أو عضوية الأسرة بوضوح.
- كل مبلغ له check يمنع القيم غير المنطقية.
- لا توجد أموال بنوع float.
- كل FK له on delete/on update مقصود.
- الحركات المالية immutable أو لها قيود تمنع تعديلها بعد الاعتماد.
- التصحيح يتم عبر reversal/adjustment.
- العمليات متعددة الخطوات داخل transaction أو function.
- كل عملية مالية مركبة atomic: نجاح كامل أو rollback كامل.
- لا توجد حالة يمكن فيها تحديث `wallets.balance` دون إنشاء ledger مطابق، أو إنشاء ledger دون تحديث الرصيد/المرجع المطلوب.
- دوال التحويل أو العمليات متعددة المحافظ تقفل الصفوف بترتيب deterministic مثل `order by id`.
- دوال `SECURITY DEFINER` تضبط `search_path` صراحة وتتحقق من صلاحيات العضو داخل الدالة.
- اختبارات SQL تغطي فشل منتصف العملية وتثبت عدم وجود partial writes.
- الفهارس تغطي RLS والاستعلامات الأساسية.
- توجد seed data للتصنيفات المصرية.

## Type Review

- الأنواع مولدة من قاعدة البيانات بعد قبول الميجريشن.
- DTOs لا تخفي حقولا مطلوبة ماليا.
- enums متطابقة مع SQL.
- لا توجد any في مسارات مالية حرجة.

## Service Review

- كل use case له وظيفة واضحة.
- التحقق من الصلاحيات ليس في الواجهة فقط.
- أخطاء العجز والقيود مترجمة لرسائل مفهومة.
- لا توجد عملية مالية جزئية عند فشل خطوة.
- الخدمات لا تنفذ العملية المالية المركبة كسلسلة طلبات منفصلة؛ تستخدم RPC atomic واحد لكل use case مالي حرج.
- retry/idempotency مغطى للعمليات المجدولة.

## UI Review

- RTL كامل.
- الرصيد الآمن يظهر أولا.
- تسجيل مصروف سريع لا يتجاوز 3 خطوات أساسية.
- رسائل الخطأ تساعد المستخدم على القرار.
- الأرقام والتواريخ والعملة بصياغة مصرية.
- الأزرار والحقول مناسبة للموبايل.
- التباين، focus، labels، keyboard، target size تمت مراجعتها ضد WCAG 2.2 AA.

## Release Review

- السيناريوهات P0 مجربة من البداية للنهاية.
- RLS تم اختباره بمستخدمين من أسرتين مختلفتين.
- لا توجد service keys في الواجهة.
- logs لا تكشف بيانات مالية حساسة.
- backup/restore أو migration reset مجرب محليا.
- المخاطر المفتوحة موثقة بقرار واضح.
