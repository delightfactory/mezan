# Complete User Flows

## 1. التهيئة الأولى

1. المستخدم ينشئ حسابا.
2. ينشئ أسرة أو يستخدم التطبيق فرديا.
3. يحدد بداية الدورة المالية: يوم القبض أو يوم يختاره.
4. يضيف المحافظ: كاش، بنك، محفظة إلكترونية، بطاقة.
5. يضيف أرصدة البداية كحركات opening balance قابلة للتدقيق.
6. يختار قوالب تصنيفات مصرية مقترحة.
7. يضيف الالتزامات المتكررة: إيجار، مدارس، أقساط، جمعية، مرافق، ديون.
8. النظام يحسب أول "رصيد آمن للصرف" ويعرضه مع تنبيهات النقص.

## 2. تسجيل دخل عادي

1. المستخدم يختار "دخل".
2. يدخل المبلغ، المصدر، المحفظة، والتاريخ.
3. يحدد هل هذا دخل الدورة الأساسية أم دخل إضافي.
4. النظام يسجل حركة دخل.
5. محرك التخصيص يقترح أو ينفذ خصم الالتزامات حسب إعدادات الأسرة.
6. تعرض النتيجة: دخل مسجل، مخصصات محجوزة، متاح للصرف.

## 3. راتب مع خصم من المنبع

1. المستخدم يختار "راتب مع خصومات".
2. يدخل إجمالي الراتب المتوقع وصافي المستلم.
3. يضيف بنود الخصم: سلفة عمل، تأمين، قسط، غياب.
4. النظام يسجل الدخل الإجمالي لأغراض التقارير.
5. يسجل حركات سداد/خصم مرتبطة بالديون أو الالتزامات.
6. يدخل فقط صافي المستلم إلى المحفظة.

## 4. مصروف سريع

1. المستخدم يضغط زر التسجيل السريع.
2. يدخل المبلغ.
3. يختار التصنيف أو يكتب وصفا يقترح منه النظام التصنيف.
4. يختار المحفظة إذا لم تكن افتراضية.
5. النظام يتحقق من الرصيد وقواعد الصرف.
6. إن كان المصروف من بند له ميزانية، يتم تحديث الاستهلاك.
7. إن تجاوز الحد، تظهر رسالة واضحة: "هتتخطى ميزانية السوبر ماركت بـ X".

## 5. تحويل بين محافظ

1. المستخدم يختار "نقل فلوس".
2. يحدد من وإلى والمبلغ.
3. النظام يسجل Transfer لا يؤثر على تقارير الدخل/المصروف.
4. لو التحويل يسحب من مخصص، يظهر تحذير عن أثره على الهدف أو الالتزام.

## 6. إنشاء مخصص مستقبلي

1. المستخدم يضيف هدفا: مدارس، طوارئ، صيانة، عيد، ملابس.
2. يدخل المبلغ المستهدف والتاريخ.
3. النظام يحسب الاستقطاع الشهري المقترح.
4. المستخدم يختار: تلقائي عند الدخل أو يدوي.
5. النظام ينشئ محفظة مخصصة مرتبطة بالهدف.

## 7. دفع التزام

1. النظام يذكّر المستخدم قبل الموعد.
2. المستخدم يختار "ادفع الآن" أو يؤجل.
3. عند الدفع، تنشأ حركة من المحفظة إلى بند الالتزام.
4. يتم تحديث حالة القسط/الدين/الجمعية.
5. لو كان المبلغ محجوزا مسبقا، يتم الدفع من المحفظة المخصصة.

## 8. الجمعية

### قبل القبض

1. المستخدم ينشئ جمعية: الاسم، عدد الشهور، القسط، شهر القبض.
2. كل قسط يسجل كتحويل إلى محفظة الجمعية.
3. تظهر الجمعية كمال مخصص/أصل منتظر.

### عند القبض

1. المستخدم يسجل قبض الجمعية.
2. النظام يدخل مبلغ القبض إلى محفظة حقيقية.
3. الرصيد المدفوع سابقا يتم تصفيته من أصل الجمعية.
4. الأقساط المستقبلية تتحول إلى التزام/دين إن كان القبض مبكرا.
5. تظهر الجمعية في مركز الالتزامات حتى نهاية الدورة.

## 9. السلف والديون

1. المستخدم يختار: "استلفت" أو "سلفت حد".
2. يدخل الطرف والمبلغ وخطة السداد.
3. عند الاستلاف، يزيد النقد وتظهر liability.
4. عند إقراض شخص، يقل النقد وتظهر receivable.
5. عند السداد، يتم تحديث الرصيد المتبقي مع حركة مالية مرتبطة.

## 10. عجز الدخل

1. النظام يحاول تخصيص الدخل حسب الأولويات.
2. يبدأ بالإجباري: إيجار، ديون، أقساط، جمعية.
3. ينتقل إلى المهم: مدارس، علاج، مرافق.
4. يؤجل أو يقلل المخصصات الأقل أولوية.
5. يعرض فجوة واضحة: "ناقص X لتغطية التزامات الشهر".
6. يقترح خيارات: تقليل ميزانية، تأجيل هدف، استخدام طوارئ، تسجيل دخل متوقع.

## 11. تصحيح حركة

1. المستخدم يفتح الحركة.
2. لا يظهر زر حذف نهائي.
3. يختار "تصحيح".
4. النظام ينشئ حركة عكسية ثم حركة جديدة بالقيمة الصحيحة.
5. يظل الأثر التاريخي قابلا للتدقيق.
