# Execution Prompt Addendum: Atomic Financial Operations

أضف هذا النص لأي أداة تنفيذ قبل بدء مرحلة الميجريشن أو الخدمات:

```text
شرط إلزامي إضافي:

كل العمليات المالية المركبة في Mezan يجب أن تكون ذرية Atomic بالكامل.

المقصود:
- إما أن تنجح العملية كاملة بكل آثارها، أو تفشل كاملة بدون أي أثر جزئي.
- ممنوع أن يتم تحديث رصيد محفظة بدون ledger transaction مطابق.
- ممنوع أن يتم إنشاء ledger transaction بدون تحديث المحافظ/الدين/الميزانية/الجمعية المرتبطة عند الحاجة.
- ممنوع تنفيذ العمليات المالية المركبة من الواجهة أو الخدمة كسلسلة استدعاءات مستقلة يمكن أن ينجح بعضها ويفشل بعضها.

يجب تنفيذ use cases المالية الحرجة كدوال PostgreSQL/Supabase RPC واحدة لكل عملية، مثل:
- تسجيل دخل مع تخصيص تلقائي.
- تسجيل مصروف وتحديث الميزانية.
- تحويل بين محافظ.
- تصحيح حركة بعكسها وإنشاء تعديل.
- قبض جمعية وتحويل المتبقي إلى دين.
- سداد دين أو قسط.

داخل كل RPC:
- تحقق من صلاحية العضو والأسرة داخل الدالة.
- استخدم transaction implicit الخاص بدالة PostgreSQL، وارفع exception عند أي فشل ليتم rollback كامل.
- اقفل الصفوف المالية المتأثرة باستخدام SELECT ... FOR UPDATE.
- عند قفل أكثر من صف، استخدم ترتيب ثابت deterministic مثل ORDER BY id لتقليل deadlocks.
- أنشئ audit event داخل نفس العملية الذرية.
- إذا كانت الدالة SECURITY DEFINER، اضبط search_path صراحة، وامنح EXECUTE فقط للأدوار المطلوبة.

اختبارات إلزامية:
- اختبار فشل منتصف العملية يثبت عدم وجود partial writes.
- اختبار تحويل متزامن يمنع الرصيد السالب.
- اختبار تصحيح حركة يثبت أن reversal والروابط والأرصدة كلها تنجح أو تفشل معا.
- اختبار قبض جمعية يثبت أن payout والرصيد والدين وحالة الجمعية كلها atomically consistent.
```
